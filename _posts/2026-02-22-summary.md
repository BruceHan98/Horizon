### [Andrej Karpathy talks about "Claws"](https://simonwillison.net/2026/Feb/21/claws/#atom-everything) ⭐️ 8.0/10

Andrej Karpathy identified 'Claws' as a new orchestration and persistence layer built on top of LLM agents, noting that multiple implementations like NanoClaw, nanobot, zeroclaw, ironclaw, and picoclaw are emerging. He specifically highlighted NanoClaw as interesting due to its minimal ~4000-line codebase that runs everything in containers by default. This matters because it represents an architectural evolution in AI systems, creating a standardized layer that handles complex orchestration, scheduling, context management, and persistence for LLM agents. The emergence of this layer could accelerate development of personal AI assistants and agentic systems by providing reusable infrastructure components. Karpathy expressed some skepticism about running OpenClaw specifically but praised the overall concept, and noted that NanoClaw's containerized approach provides filesystem isolation rather than just permission checks. The 'Claw' terminology is becoming established for AI agent systems that run on personal hardware, communicate via messaging protocols, and can both act on direct instructions and schedule tasks.

*Source: rss/Simon Willison*

**Background**: LLM agents are AI systems that use large language models to reason, plan, and execute tasks by calling tools or APIs. Orchestration refers to the coordination and management of these agents' workflows, including scheduling tasks and maintaining context across interactions. Persistence in this context means maintaining state, memory, or context between different agent executions or sessions.

### [Taalas serves Llama 3.1 8B at 17,000 tokens/second](https://simonwillison.net/2026/Feb/20/taalas/#atom-everything) ⭐️ 8.0/10

Canadian hardware startup Taalas announced its first product, a custom hardware implementation called Silicon Llama that runs Meta's Llama 3.1 8B model at a speed of 17,000 tokens per second. This performance is achieved through aggressive quantization techniques that combine 3-bit and 6-bit parameters. This represents a major breakthrough in efficient AI deployment, as such high inference speeds could enable real-time, low-latency applications for a popular open-source model that were previously impractical. It signals a shift towards specialized hardware that could make powerful LLMs more accessible and cost-effective to run at scale. The model is described as 'aggressively quantized,' using a mixed-precision approach with 3-bit and 6-bit parameters, and the company has indicated its next generation will use 4-bit quantization. A live demo of the technology is available for testing at chatjimmy.ai.

*Source: rss/Simon Willison*

**Background**: Quantization is a technique that reduces the precision of a neural network's parameters (like weights and activations) from higher bit-widths (e.g., 32-bit) to lower ones (e.g., 8-bit, 4-bit, or lower) to decrease model size and computational requirements, often at a potential cost to accuracy. Aggressive quantization pushes this process further, using very low bit-widths like 3-bit or 4-bit, which typically requires specialized techniques or hardware to maintain acceptable performance. Custom hardware for LLM inference, like Taalas's Silicon Llama, is designed specifically to run these quantized models efficiently, offering potential speed and power advantages over general-purpose GPUs. Llama 3.1 8B is a 8-billion parameter open-source language model released by Meta in July 2024.

*Source: hackernews/vinhnx*

The 'Plan-and-Execute' pattern is a recognized agentic AI pattern where a comprehensive planning phase is separated from a distinct execution phase to improve reliability and reduce errors in complex tasks.

**Discussion**: The community strongly endorses the planning-first approach, with commenters noting its effectiveness and reliability across different skill levels and its similarity to how senior engineers work. ### [Quoting Thariq Shihipar](https://simonwillison.net/2026/Feb/20/thariq-shihipar/#atom-everything) ⭐️ 6.0/10

Thariq Shihipar, a developer at Anthropic, explained that the long-running AI agent product Claude Code is fundamentally enabled by prompt caching, which reuses computation from previous interactions to significantly decrease latency and cost. He revealed that Claude Code's entire system harness is built around this technique, and they monitor prompt cache hit rates as a critical production metric, declaring SEV (Severity) alerts when rates drop too low. This insight matters because it reveals a key technical and economic enabler for commercially viable, long-running AI agents, which are otherwise prohibitively expensive and slow. The approach directly impacts product feasibility, allowing for more generous user rate limits in subscription plans and making complex, multi-step AI coding assistants like Claude Code practical to operate at scale. A high prompt cache hit rate is so critical to the system's economics and performance that it is treated as a core operational health metric, with low rates triggering SEV-level production incidents. The entire Claude Code application harness is architecturally designed around this caching mechanism, indicating it's not just an optimization but a foundational component.

*Source: rss/Simon Willison*

**Background**: Prompt caching is a technique where the intermediate computational results or outputs from a large language model (LLM) for a given input (prompt) are stored and reused if the same or a similar prompt is encountered again, avoiding redundant processing. 'Agentic' AI products refer to systems where an AI agent can perform a sequence of actions or steps autonomously to achieve a goal, such as writing or debugging code over multiple turns. In production software engineering, a SEV (Severity) alert is a high-priority incident notification triggered when a key system metric falls outside acceptable bounds, requiring immediate attention from on-call engineers.

### [ggml.ai joins Hugging Face to ensure the long-term progress of Local AI](https://simonwillison.net/2026/Feb/20/ggmlai-joins-hugging-face/#atom-everything) ⭐️ 8.0/10

Georgi Gerganov's ggml.ai organization, the creator of the influential llama.cpp project, announced it is joining Hugging Face to advance the development of local AI models, with the stated goals of achieving seamless integration with the Transformers library and improving the packaging and user experience of ggml-based software. This merger is significant because it unites a foundational project for local AI inference (llama.cpp) with the leading open-source AI platform (Hugging Face), which could streamline the process for model developers to release models compatible with local, efficient inference from the start, accelerating the growth of the local AI ecosystem. The announcement specifies that the projects will remain open source and the community autonomous, with Georgi and his team continuing to maintain everything, and the initial technical focus will be on integration with Hugging Face's Transformers library and improving the user experience for deploying local models.

*Source: rss/Simon Willison*

**Background**: llama.cpp is an open-source C/C++ library that enables efficient inference of large language models (LLMs) on consumer hardware, famously using techniques like 4-bit quantization to drastically reduce memory requirements. Its release in March 2023 was pivotal, as it allowed models like Meta's LLaMA to run on standard laptops and MacBooks without specialized NVIDIA GPUs, democratizing local AI. Hugging Face is a major AI platform and the steward of the widely-used Transformers library, which serves as a standard framework for defining and using AI models. The ggml library is the underlying tensor library developed alongside llama.cpp, designed for high-performance machine learning on commodity hardware.
